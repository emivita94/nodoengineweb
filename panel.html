Integración con ERP
Guía técnica completa — todo lo que el desarrollador necesita para integrarse
🔑
Credenciales de conexión
Estos son los únicos dos datos que el desarrollador necesita para empezar
URL BASE DE LA API
—
Todos los endpoints se construyen sobre esta URL. Por ejemplo, para emitir una factura: URL_BASE + /documentos/erp
API KEY — HEADER DE AUTENTICACIÓN
—
Esta clave identifica al emisor (empresa). Se envía en cada request como header HTTP: X-API-Key: sk_live_XXXX.
Sin este header, la API devuelve 401 Unauthorized. La clave es secreta — no exponerla en el frontend ni en logs.
⚠️ Ambiente de pruebas vs producción
Las keys que empiezan con sk_test_ operan contra el ambiente de pruebas de SIFEN (no envían documentos reales a la SET).
Las keys sk_live_ operan en producción — cada documento enviado queda registrado oficialmente en la SET.
Usá sk_test_ para todas las pruebas de integración antes de pasar a producción.
