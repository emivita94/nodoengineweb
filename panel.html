¿Cómo funciona la integración?
1
Tu ERP genera una venta → hace un POST a NODO Engine con los datos de la factura
2
NODO Engine valida los datos, firma el XML con el certificado digital y lo envía a la SET
3
SET aprueba o rechaza → NODO devuelve el resultado al ERP y opcionalmente notifica por webhook
4
Tu ERP descarga el KUDE (PDF) o XML y lo entrega al cliente
Autenticación: todos los requests deben incluir el header X-API-Key: sk_live_XXXX. Sin él, la API devuelve 401 Unauthorized.

Descargar KUDE (PDF) y XML
Solo disponible para documentos con estado aprobado
Descargar KUDE en PDF
GET /api/v1/documentos/{cdc}/kude?formato=a4 
// Devuelve el PDF binario (Content-Type: application/pdf)
// formato: a4 (predeterminado) | carta | ticket

// Ejemplo en tu sistema:
response = GET /api/v1/documentos/01800.../kude?formato=a4
           X-API-Key: sk_live_XXXX
// → guardar como archivo .pdf o mostrar en el navegador
Descargar XML firmado
GET /api/v1/documentos/{cdc}/xml 
// Devuelve el XML firmado (Content-Type: application/xml)
// Este es el archivo oficial para presentar ante la SET
Importante: estos endpoints requieren el header X-API-Key igual que el resto. No son URLs públicas directas.
Recibir notificaciones por Webhook
Si enviás webhookUrl en el request, NODO notifica a tu sistema cuando el documento es procesado
NODO hace POST → tu-erp.py/webhook/nodo 
{
  "evento": "de.aprobado",
  "cdc": "01800123456001002000001200120251234567890",
  "numero": "001-002-0000001",
  "tipo": 1,
  "estado": "aprobado",
  "sifen_codigo": "0260",
  "referenciaExterna": "FAC-2025-001",
  "timestamp": "2025-04-03T10:30:45Z"
}
Eventos posibles
EventoDescripción
de.aprobadoDocumento aprobado por la SET — listo para entregar al cliente
de.rechazadoDocumento rechazado — revisar sifen_codigo para el motivo
de.canceladoDocumento cancelado exitosamente
Tu servidor debe responder HTTP 200 para confirmar la recepción. Si no responde, NODO reintenta hasta 3 veces con intervalos de 1 minuto.

CampoTipoReq.Descripción

Guía de integración
Cómo configurar el ERP del cliente paso a paso
1
Configurar el ERP con la URL y API Key
En el ERP del cliente (Qualita, SAP, etc.) buscar la sección de "Facturación electrónica" o "SET" y cargar la URL del endpoint y la API Key que generaste arriba.
2
El ERP envía cada factura con este formato
POST /api/v1/documentos/erp 
{
  "tipoDocumento": 1,
  "fecha": "2025-04-03T10:30:00",
  "moneda": "PYG",
  "cliente": {
    "tipo": 1,
    "documento": "4567890",
    "razonSocial": "Juan Pérez"
  },
  "items": [
    {
      "descripcion": "Producto A",
      "cantidad": 2,
      "precioUnitario": 75000,
      "precioTotal": 150000,
      "tasaIVA": 10
    }
  ],
  "referenciaExterna": "FAC-2025-001",
  "webhookUrl": "https://mi-erp.py/webhook/nodo"
}
3
La API responde con el resultado
HTTP 201 Created 
{
  "ok": true,
  "data": {
    "cdc": "01800123456001001000001200120251234567890",
    "numero": "001-001-0000001",
    "estado": "aprobado",
    "sifen_codigo": "0260",
    "sifen_mensaje": "Documento aprobado",
    "kude_url": "/api/v1/documentos/01800.../kude",
    "xml_url": "/api/v1/documentos/01800.../xml"
  }
}
4
Si configuraste webhookUrl, el ERP recibe una notificación
POST → tu-erp.py/webhook/nodo 
{
  "evento": "de.aprobado",
  "cdc": "01800123456001001000001200120251234567890",
  "numero": "001-001-0000001",
  "estado": "aprobado",
  "referenciaExterna": "FAC-2025-001",
  "timestamp": "2025-04-03T10:30:45Z"
}
Con la referenciaExterna el ERP sabe exactamente a qué factura interna corresponde.
Errores comunes y cómo resolverlos
401 API Key inválida o revocada — verificar el header X-API-Key
422 Datos inválidos — revisar formato RUC, montos, tasaIVA (0, 5 ó 10)
0300 SET rechazó el documento — revisar logs para el detalle del campo con error
cert Certificado vencido — renovar en Configuración → Certificado digital
